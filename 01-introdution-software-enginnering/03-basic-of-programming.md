# Basic of programming

## Comparing Compiled and Interpreted Programming Languages

### Interpreted Programming Languages

- Interpreted languages run source code through an interpreter line by line, often built into operating systems or web browsers, making them suitable for smaller, repetitive tasks.
- Examples include Python, JavaScript, and HTML, which are easier to learn, cross-platform, and commonly used for scripting and web development.

### Compiled Programming Languages

- Compiled languages translate source code into executable files that run directly on a device, making them ideal for larger, complex programs requiring faster performance.
- Examples include C, C++, C#, and Java, which are case-sensitive, more difficult to learn, and used for applications like operating systems and spreadsheet programs.

### Key Differences and Use Cases

- Interpreted languages are platform-independent and better for automation and web tasks, while compiled languages are platform-specific and suited for resource-intensive applications.
- Developers choose interpreted languages for quick, smaller tasks and compiled languages for robust, high-performance software.

## Query and Assembly Programming Languages

### High-Level vs Low-Level Programming Languages

- High-level languages use English-like syntax to simplify coding and debugging; examples include SQL, Pascal, and Python.
- Low-level languages use symbolic representations of machine code and are closely tied to hardware architecture; assembly languages like ARM, MIPS, and X86 are examples.

### Query Languages

- A query language is used to request or manipulate data in databases, with SQL being the most common.
- Queries can be select commands to retrieve data or action commands (CREATE, INSERT, UPDATE) to modify data, often referred to as CRUD operations.

### Assembly Languages

- Assembly languages represent machine code with simple symbols and are specific to processor architectures.
- They are translated by assemblers, with each assembly instruction corresponding to one machine code instruction, unlike high-level languages where one statement can translate to many machine instructions.

## Understanding Code Organization Methods

### Importance of Code Organization

- Organizing code before programming leads to cleaner, more reliable code and reduces bugs.
- Well-planned software design provides a consistent and logical format for coding.

### Methods of Code Organization

- Two main methods are flowcharts and pseudocode.
- Flowcharts visually represent algorithms with symbols and arrows, useful for smaller problems.
- Pseudocode is a high-level, language-independent description of an algorithm, better suited for larger projects.

### Flowcharts

- Use standard symbols like capsules (start/end), rectangles (process), diamonds (decision), and parallelograms (data).
- Help analyze problem-solving methods and communicate logic easily.
- Software tools like Microsoft Visio and Lucidchart assist in creating flowcharts.

### Pseudocode

- Informal programming description without strict syntax, bridging programmer logic and actual code.
- Simplifies focus on logic rather than language syntax, making it easier for beginners.
- Facilitates communication among programmers and non-programmers and is concise and easy to modify.

Overall, organizing code with flowcharts or pseudocode enhances software quality and development efficiency.

## Branching and Looping Programming Logic

### Programming Logic Types

- Branching logic allows a program to make decisions and follow different instruction paths based on conditions using Boolean expressions (true/false).
- Looping logic involves repeating a sequence of instructions until a specific condition is met, enabling repeated actions.

### Boolean Expressions and Variables

- Boolean expressions evaluate to either true or false and are used to control program flow.
- Variables hold values that can change during program execution and are passed into functions or subroutines.

### Branching Constructs

- Common branching statements include if, if-then-else, switch, and GoTo, which direct the program to different code paths based on conditions.
- Branching enables complex decision-making by creating multiple branches depending on parameter values.

### Looping Constructs

- Three basic loop types are While (condition checked before loop body), For (counter-controlled loop), and Do-while (condition checked after loop body).
- Loops repeat instructions until a condition is met, then continue with the next part of the program.

In summary, branching decides what actions to take based on conditions, while looping controls how many times actions are performed, both relying on Boolean logic and variables.
