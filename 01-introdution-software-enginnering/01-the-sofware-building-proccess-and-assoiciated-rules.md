## Software Testing

### Types of Software Testing

- Functional testing verifies that the software meets functional requirements by testing inputs and expected outputs, often using black-box methods.
- Non-functional testing assesses attributes like performance, security, scalability, and availability to ensure the software behaves properly under various conditions.
- Regression testing ensures that recent changes or bug fixes do not negatively impact existing functionality by re-running selected test cases.

### Levels of Software Testing

- Unit testing checks individual code sections, usually done by developers during development to catch errors early.
- Integration testing examines interactions between combined code modules to find bugs in their communication or logic.
- System testing evaluates the complete integrated system against specified requirements, covering both functional and non-functional aspects.
- Acceptance testing validates that the system meets user needs and business processes, typically performed by customers or stakeholders.

## Documentation

### Documentation Formats and Audiences

- Software documentation can be written, video, or graphical and serves various audiences including end users, developers, QA engineers, and system administrators.
- It is applicable throughout all phases of the Software Development Life Cycle (SDLC).

### Types of Documentation

- Product documentation focuses on the software’s functionality and includes requirements, design, technical, quality assurance, and user documentation.
- Process documentation describes how to complete tasks and includes standard operating procedures (SOPs) that provide detailed, step-by-step instructions specific to an organization.

### Purpose and Maintenance of Documentation

- Documentation helps explain what the software is and how to use it, supporting development, testing, and user operation.
- It must be kept up to date, especially during the maintenance phase of the SDLC, to ensure accuracy and relevance.

## Roles in Software Engineering Projects

### Common Roles in Software Engineering Projects

- Project Manager or Scrum Master: Oversees project planning, scheduling, budgeting, resource allocation, and team communication; Scrum Master focuses on facilitating team success and communication in Agile environments.
- Stakeholders: Include customers, end-users, and decision-makers who define project requirements and provide feedback, sometimes participating in testing phases.

### Technical and Design Roles

- System/Software Architect: Designs and communicates the software architecture, providing technical support throughout the development lifecycle.
- UX Designer: Focuses on user experience by making software intuitive and robust, defining how users interact with the software.

### Development and Quality Assurance Roles

- Software Developer: Writes code implementing architecture, requirements, and UX designs.
- Tester/QA Engineer: Ensures software quality by writing and executing test cases to identify bugs and provide feedback.

### Operations and Product Management Roles

- Site Reliability/Operations Engineer: Combines software engineering and IT management to ensure system reliability, automate processes, and troubleshoot issues.
- Product Manager/Owner: Holds the product vision, understands client and user needs, and leads development to deliver stakeholder value.

### Documentation Role

- Technical Writer/Information Developer: Creates user documentation to help end-users and customers understand and use the software effectively.

## Summary

### Software Engineering Fundamentals

- Software engineering applies scientific principles to design, build, and maintain software systems.
- The Software Development Life Cycle (SDLC) improves efficiency and communication among team members and stakeholders.

### Software Development Processes and Documentation

- Key processes include requirements gathering, design, coding, testing, releasing, and documenting.
- Different requirement documents include SRS (Software Requirements Specification), URS (User Requirements Specification), and SysRS (System Requirements Specification).

### Development Methodologies and Testing

- Common SDLC methodologies are Waterfall, V-shape, and Agile.
- Testing types include functional testing (input-output validation), non-functional testing (performance, security), and regression testing (ensuring changes do not break existing features).

### Roles in Software Engineering Projects

- Roles include project manager/scrum master, stakeholders, architects, UX designers, developers, testers, operations engineers, product managers, and technical writers.
- Each role contributes uniquely to the software development and delivery process.
